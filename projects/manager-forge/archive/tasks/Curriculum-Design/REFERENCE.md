## Workflow

1. **Get Module BRIEF** (provided by PM or fetch from roadmap)
2. **Use SYSTEM.md** for tone, output rules, examples
3. **Use _TEMPLATES/** to scaffold BRIEF + SCRIPT + LEARNER-PACK
4. **Output:** BRIEF.md, MODULE-SCRIPT.md, LEARNER-PACK.md (+ RESEARCH.md if applicable)
5. **Update STATUS.md** when module is complete

## Output Rules (From SYSTEM.md)

- **Module Script:** 3–5 sub-modules, 2,000–3,000 words per sub-module, video-ready
- **Learner Pack:** 1–2 worksheets, 1 self-assessment, 2–3 role-plays, 4–5 Zoom prompts per sub-module
- **Tone:** Direct, empathetic, conversational, battle-tested (no corporate fluff)
- **Must include:** Real examples, actionable frameworks, honest acknowledgment of difficulty, vulnerability
