## What Was Built

Each module has two documents:

**LEARNER-PACK.md** — The async companion content. Worksheets, self-assessments, role-plays, discussion prompts, and a quick reference card. Format matches module-03 (which is the established bar). Each pack contains:
- 5–6 worksheets with real fill-in exercises, not open-ended reflections
- 2–5 self-assessments with scoring and interpretation
- 3–8 role-play scenarios with setup, your-turn prompts, and example scripts
- Discussion prompts for cohort peer exchange
- Quick reference card with the module's core frameworks

**AI-LAYER.md** — Net-new companion document (no archive equivalent). Covers:
- What the manager's job looks like in this topic area now vs. before AI
- 4–5 new failure modes with specific tells and costs
- 2–4 concrete scenarios with wrong moves, right moves, and example language
- How each module's original material adapts (what's unchanged, what's updated)

## Judgment Calls Made — Andrew Should Review These

**1. Module 02 AI Layer — The "VALIDATE" step**
Added a fourth step to the Inform-Advocate-Execute framework specifically for AI-generated forecasting data. Feels consistent with the spirit of the framework, but it's an extension Andrew hasn't explicitly signed off on. He should check whether this framing works given how he teaches the original in sessions.

**2. Module 04 — Framing of "two grief cycles in parallel"**
The AI Layer describes a second grief cycle (the parts of the manager job that AI now does) running alongside the original IC-to-manager grief cycle. This is a meaningful conceptual extension of the original module. If Andrew doesn't connect with the "two grief cycles" framing, it can be simplified. But given how grounded the original grief cycle content is, this felt like the right metaphor.

**3. Module 07 LEARNER-PACK — New worksheets vs. archive worksheets**
The archive module-07 LEARNER-PACK has strong pipeline map worksheets and conversion rate exercises. The new LEARNER-PACK maintains all the foundational math and diagnostic exercises, but adds AI-specific worksheets (validating the AI forecast, coaching from call analysis, stage inflation audit). If any specific archive worksheets by name (WS-07-xx) were expected to be ported verbatim, that would need a pass-through comparison.

**4. Module 09 — AI fluency weighting in the hiring rubric**
The AI Layer describes AI fluency as a current ~30% weight that should drop over time as it becomes table stakes. This is an educated guess at appropriate weighting given current market conditions — Andrew's actual hiring practice at TrackForce should calibrate this number.

**5. Module 10 — "The Skeptical Rep Who's Poisoning the Team" scenario**
The Doug scenario in the AI Layer walks through a direct 1:1 conversation where the manager draws a line between Doug's personal skepticism (acceptable) and his team-level commentary (not acceptable). This is a judgment call on a culturally sensitive scenario. Andrew should verify this framing feels right as his voice.

**6. Module 11 — Automation maintenance as a new time cost**
The AI Layer calls out that AI automation requires ongoing maintenance that manual workflows didn't — prompts drift, model updates change outputs, input data changes. This is accurate but adds a nuance the original module doesn't have. Worth checking that Andrew agrees with this framing before teaching it.
